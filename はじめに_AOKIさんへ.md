# ネットワーク詳細図ツール ― 使い方ガイド（AOKIさんへ）

このガイド通りに上から順番にやれば、誰でも動かせるように書いています。
所要時間は最初のセットアップで **15〜30分くらい**。2回目以降は1ステップで起動できます。

---

## このツールでできること

- ノード（人・物・場所・道具・行為）と、それらを結ぶ線（関係性）を、画面上でドラッグ操作で組み立てられる
- 「施主が敷地のアカマツを切って梁にする」みたいな日本語を打ち込むと、AIが自動でノードと線を生成してくれる
- 詳細図 / 全体図 / 時間軸図 を切り替えて表示できる

---

## 全体の流れ

1. **Node.js をインストール**（PCで一度だけ）
2. **プロジェクトのフォルダを受け取る**
3. **Gemini API キーを取得**（無料・5分くらい）
4. **API キーをファイルに書き込む**
5. **必要な部品を一括インストール**（一度だけ）
6. **アプリを起動**
7. **ブラウザで開いて使う**

---

## ステップ 1：Node.js をインストール

「Node.js（ノードジェイエス）」というソフトを入れます。これがないとアプリが動きません。

1. ブラウザで **https://nodejs.org/** を開く
2. **「LTS」版**（左側、緑のボタン）をダウンロード
3. ダウンロードしたインストーラーを実行 → 全部「次へ」「同意する」でOK
4. インストール完了

**確認のしかた**

- Windows: スタートメニューで「PowerShell」または「コマンドプロンプト」を開く
- Mac: 「ターミナル」を開く（Spotlight で `terminal` と打つ）

開いた黒い画面に以下を打ってEnter：

```
node -v
```

`v20.x.x` のような数字が表示されればOKです。

---

## ステップ 2：プロジェクトのフォルダを受け取る

俊貴から `network_design` というフォルダ（ZIPファイル）を受け取ります。

1. ZIPをダウンロードしたら **解凍**（ダブルクリック or 右クリック→展開）
2. デスクトップなど、自分が分かりやすい場所に置く
   - 例：`デスクトップ/network_design/`
3. フォルダの中に `web` というフォルダがあるはずです。これをこの後使います。

---

## ステップ 3：Gemini API キーを取得（無料）

AI生成機能を使うために、Googleが配布している無料の「APIキー」というパスワードのようなものを取得します。

1. ブラウザで **https://aistudio.google.com/** にアクセス
2. Googleアカウントでログイン（普段使っているGmailのアカウントでOK）
3. 左メニューから **「Get API key」** または **「APIキーを取得」** をクリック
4. **「Create API key」** または **「APIキーを作成」** ボタンを押す
5. 表示される長い文字列（`AIza...` で始まる）をコピー
   - ⚠️ この文字列が **APIキー本体**。これを次のステップで使います
   - ⚠️ パスワードと同じ扱い。他人に見せない・SNSに貼らない

---

## ステップ 4：API キーをファイルに書き込む

ここが少しだけ難しいパートです。**`.env.local`（ドットエンブドットローカル）** というファイルを新しく作って、そこにAPIキーを書き込みます。

### Windowsの場合

1. 受け取ったフォルダの中の **`web` フォルダを開く**
2. フォルダ内の何もない場所で **右クリック → 新規作成 → テキストドキュメント**
3. ファイル名を **そのまま全部** `.env.local` に書き換える
   - ⚠️ `新しいテキストドキュメント.txt` ではなく、`.env.local` にする
   - 「拡張子を変更しますか？」と聞かれたら **「はい」**
   - 拡張子（`.txt` など）が表示されない場合は、エクスプローラの「表示」メニュー →「ファイル名拡張子」にチェックを入れてからやり直す
4. 作った `.env.local` を **メモ帳** で開く（右クリック →「プログラムから開く」→ メモ帳）
5. 以下を貼り付ける（`AIza...` の部分はステップ3でコピーしたAPIキー本体に置き換える）：

   ```
   VITE_GEMINI_API_KEY=AIzaここに自分のキーを貼る
   ```

6. 保存して閉じる

### Macの場合

1. `web` フォルダを開く
2. メニューバー：**Finder → 表示 → 表示オプションを表示** → 「すべてのファイル名拡張子を表示」をON
3. テキストエディット.app を開く → メニュー：**フォーマット → 標準テキストにする**
4. 以下を入力（`AIza...` の部分はステップ3でコピーしたAPIキー本体に置き換える）：

   ```
   VITE_GEMINI_API_KEY=AIzaここに自分のキーを貼る
   ```

5. **ファイル → 保存** → ファイル名 `.env.local`（先頭のドット忘れずに）→ 場所は `web` フォルダ
6. 「ドットから始まるファイル名は…」と警告が出たら **「ドットを使用」** を選ぶ

### ✅ 確認

`web` フォルダの中に `.env.local` というファイル（拡張子なし、先頭がドット）ができていればOK。

---

## ステップ 5：必要な部品を一括インストール（最初だけ）

ターミナル（Windowsなら PowerShell、Macなら ターミナル）を開いて、`web` フォルダに移動します。

### 一番簡単な方法：「フォルダから直接ターミナルを開く」

- **Windows**: `web` フォルダを開いて、上のアドレスバーをクリック → `powershell` と打ってEnter
- **Mac**: `web` フォルダを Finder で右クリック → **「フォルダに新規ターミナル」**（出ない場合：システム設定 → キーボード → ショートカット → サービス で有効化）

開いた黒い画面に以下を打ってEnter：

```
npm install
```

数分かかります。緑色の文字でメッセージが出て、最後にプロンプト（`PS C:\...>` や `$`）が戻ってきたら完了。

⚠️ エラーが出ても落ち着いて。たいていは **もう一度 `npm install` を打つ** か、**Node.jsをインストールし直す** で解決します。

---

## ステップ 6：アプリを起動

同じターミナルで、続けて以下を打ってEnter：

```
npm run dev
```

しばらくすると以下のような表示が出ます：

```
  VITE v8.0.10  ready in 354 ms

  ➜  Local:   http://localhost:5173/
```

これが **「アプリが起動した」** サイン。
このターミナルの黒い画面は **閉じない** でください（閉じるとアプリも止まります）。

---

## ステップ 7：ブラウザで使う

1. ブラウザで **http://localhost:5173/** を開く
2. 画面が出たら成功

### 基本の操作

| やりたいこと | 操作 |
|---|---|
| ノードを追加 | 画面の何もないところを **ダブルクリック** / 上のツールバー「＋ノード追加」 |
| 線（エッジ）を引く | ノードの中心の白い丸から、別のノードへ **ドラッグ** |
| ノードを動かす | ドラッグでそのまま動かせる |
| 詳細図を切り替え | 上のタブで切り替え |
| AIで自動生成 | 画面右下の **「✨ AI生成」** をクリック |

### AI生成パネルの使い方

1. 右下の「✨ AI生成」を開く
2. テキスト欄に日本語で関係性を書く
   - 例：「施主が敷地のアカマツを切り、敷地内で乾燥させ、チョウナではつって、家の梁として使う。」
3. 「ノード/エッジを生成」ボタンを押す
4. 数秒待つと、ノードと線が自動で増える

⚠️ ステップ4で `.env.local` にキーを書き込んでいれば、APIキー入力欄は表示されません。
逆にAPIキー入力欄が出ている場合は、`.env.local` がうまく読まれていない可能性が高いです（→ 下の「困ったとき」を参照）。

---

## 終了するとき

- ブラウザのタブを閉じる
- ターミナルの黒い画面で **`Ctrl + C`** を押す（Macも `Ctrl + C`）→ アプリが止まる
- ターミナルを閉じる

---

## 2回目以降の起動（簡単）

- ターミナルを開く
- `web` フォルダに移動
- `npm run dev` を打つ
- ブラウザで `http://localhost:5173/` を開く

これだけ。`npm install` も `.env.local` の作成もやり直しは不要です。

---

## 困ったとき

### `npm` が見つからないと言われる
→ Node.js のインストールが完了していない、またはターミナルを再起動していない。
ターミナルを一度閉じて、新しく開き直してから試してください。

### `npm install` でエラーが出る
→ ネット接続を確認 → もう一度 `npm install` を打つ。
それでも直らないなら、`web` フォルダ内の `node_modules` フォルダと `package-lock.json` を削除してから `npm install` をやり直す。

### ブラウザに「このページにアクセスできません」と出る
→ ターミナルの黒い画面に `ready in xxx ms` の表示があるか確認。
無ければ起動が失敗している。エラーメッセージを俊貴に送ってください。

### AI生成パネルにAPIキー入力欄が表示される（消えない）
→ `.env.local` の置き場所か中身がおかしい。チェック：
- ファイルが **`web` フォルダ直下** にある（`web/src/` の中ではない）
- ファイル名が **完全に `.env.local`**（`.env.local.txt` になっていない）
- 中身が **`VITE_GEMINI_API_KEY=AIza...`** の1行（前後に余計なスペース・引用符なし）
- 直したら **ターミナルで `Ctrl + C` → もう一度 `npm run dev`** で再起動

### AI生成でエラーが出る
→ APIキーが間違っている、またはGoogle側のAPIが有効化されていない可能性。
Google AI Studio (https://aistudio.google.com/) でキーを再発行して `.env.local` を書き換えてみてください。

---

## ⚠️ APIキーの取り扱い

- `.env.local` ファイルを **他人に共有しない**（メール添付・チャットへの貼り付けNG）
- フォルダごと誰かに渡すときは、**`.env.local` だけ削除してから** 渡す
- もし誤ってどこかに貼ってしまった場合は、Google AI Studio でキーを **削除して新しいキーを作り直す**
